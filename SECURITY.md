# Security Policy

## Reporting a Vulnerability

The Balancer team takes security seriously. We appreciate your efforts to responsibly disclose your findings.

### How to Report

If you discover a security vulnerability, please report it through one of the following channels:

- **Immunefi Bug Bounty Program**: https://immunefi.com/bounty/balancer/
- **Email**: security@balancer.fi

**Please do NOT:**
- Open a public GitHub issue for security vulnerabilities
- Disclose the vulnerability publicly before it has been addressed

### What to Include

When reporting a vulnerability, please include:

- A clear description of the vulnerability
- Steps to reproduce the issue
- Affected contract addresses and networks
- Potential impact of the vulnerability
- Any suggested fixes (if available)

### Response Timeline

- **Initial Response**: Within 48 hours
- **Status Update**: Within 7 days
- **Resolution Target**: Depending on severity

### Scope

This security policy applies to:

- Smart contracts deployed through this repository
- Deployment scripts and infrastructure
- Contract artifacts and build information

### Bug Bounty Program

Balancer maintains a bug bounty program on Immunefi. Please refer to the program page for:

- Reward amounts based on severity
- In-scope and out-of-scope items
- Eligibility requirements

Visit: https://immunefi.com/bounty/balancer/
